# Team Onboarding Checklist

Use this checklist to decide whether a new contributor can start in the repository without guessing.

---

## Start here path

- [ ] There is one obvious root starting point
- [ ] The learning or reading order is documented
- [ ] New contributors can tell which file to open first

## Source-of-truth docs

- [ ] Project guidance files are easy to find
- [ ] Present facts are separate from future plans
- [ ] Missing tooling or missing structure is labeled clearly

## Safe-to-copy assets

- [ ] Templates are listed in one obvious place
- [ ] Each template says what it is for
- [ ] Each template says how it should be adapted locally

## Shared expectations

- [ ] Contributors are told not to invent commands or frameworks
- [ ] Contributors are told to prefer small additive changes
- [ ] Contributors are told to update guidance when reality changes

## Final check

- [ ] A new contributor can start without guessing
- [ ] A new contributor can tell what is real now vs not yet present
