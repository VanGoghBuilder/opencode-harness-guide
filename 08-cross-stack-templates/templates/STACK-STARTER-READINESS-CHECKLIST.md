# Stack Starter Readiness Checklist

Use this checklist before adding a stack-specific starter kit.

---

## Scope check

- [ ] The stack is actually part of repository scope
- [ ] The starter would add value beyond the shared core docs

## Reality check

- [ ] The package manager is verified by real files
- [ ] Build command is verified or intentionally marked `TBD`
- [ ] Lint command is verified or intentionally marked `TBD`
- [ ] Test command is verified or intentionally marked `TBD`
- [ ] Single test command is verified or intentionally marked `TBD`

## Teaching check

- [ ] The file layout is stable enough to teach
- [ ] The starter would not hide important project-specific choices
- [ ] The stack-specific guidance would not contradict the shared beginner path

## Boundary check

- [ ] Universal guidance is already covered in the shared modules
- [ ] Stack-specific details are clearly separated from universal patterns
- [ ] Unknown details are still marked `TBD` instead of guessed

## Final outcome

- [ ] Ready for a stack-specific starter kit
- [ ] Not ready yet; keep the guidance in the shared core
