# Advanced Workflow Checklist

Use this to decide whether a repeated multi-step process is ready to become part of your harness.

Use this checklist before turning a repeated process into a multi-step workflow.

---

## Workflow shape

- [ ] The workflow goal is described in one sentence
- [ ] The start condition is clear
- [ ] The success condition is clear
- [ ] The workflow has a defined stopping point

## Facts-first safety

- [ ] Repository reality is checked before planning
- [ ] Missing commands or tooling are still marked `TBD`
- [ ] Future direction is not written as if it already exists

## Coordination points

- [ ] Handoff points are clear
- [ ] Review or validation points are clear
- [ ] Related root docs will be updated if needed
- [ ] Related templates will be updated if needed

## Scope control

- [ ] The workflow is not wider than the repository can currently support
- [ ] The workflow can be explained to a new contributor
- [ ] The workflow does not depend on undocumented conventions

## Final decision

- [ ] Ready to scale into a repeatable workflow
- [ ] Not ready yet; keep it smaller and more local
