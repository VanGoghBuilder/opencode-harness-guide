# Support

Use this file to decide where a question, bug report, docs issue, or sensitive report should go.

---

## If you have a normal public docs or repo issue

Use the public issue templates:
- [.github/ISSUE_TEMPLATE/bug_report.md](.github/ISSUE_TEMPLATE/bug_report.md)
- [.github/ISSUE_TEMPLATE/feature_request.md](.github/ISSUE_TEMPLATE/feature_request.md)
- [.github/ISSUE_TEMPLATE/documentation.md](.github/ISSUE_TEMPLATE/documentation.md)
- [.github/ISSUE_TEMPLATE/question.md](.github/ISSUE_TEMPLATE/question.md)

Use these for:
- documentation problems
- navigation drift
- broken examples
- feature requests for the harness guide
- normal questions about repo structure

---

## If the issue is sensitive

Do not post details publicly first.
Read:
- [SECURITY.md](SECURITY.md)
- [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
- [.github/SECURITY_REPORTING.md](.github/SECURITY_REPORTING.md)

---

## What this repo cannot do yet

This repository does **not** yet publish a dedicated private support, security, or conduct reporting channel.

That means:
- public issue templates are fine for normal questions
- sensitive reports should not be posted with full details
- some reporting flows are documented here, but not fully completed in infrastructure yet
