# Support

This repository is a documentation-first project, so “support” here mostly means help with repository contents, contribution flow, issue routing, and reporting boundaries.

---

## What to use for normal public help

Use the public issue templates for:

- documentation problems
- factual inconsistencies in the repo
- feature requests for the docs/templates rewrite
- beginner questions about how this repository is organized

Relevant files:

- [.github/ISSUE_TEMPLATE/bug_report.md](.github/ISSUE_TEMPLATE/bug_report.md)
- [.github/ISSUE_TEMPLATE/feature_request.md](.github/ISSUE_TEMPLATE/feature_request.md)
- [.github/ISSUE_TEMPLATE/documentation.md](.github/ISSUE_TEMPLATE/documentation.md)
- [.github/ISSUE_TEMPLATE/question.md](.github/ISSUE_TEMPLATE/question.md)

---

## What not to post publicly

Do **not** post these in a normal public issue:

- real secrets or tokens
- sensitive security problems
- private personal details tied to a conduct report

For those cases, first read:

- [SECURITY.md](SECURITY.md)
- [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
- [.github/SECURITY_REPORTING.md](.github/SECURITY_REPORTING.md)

---

## Current limitation

This repository does **not yet publish a dedicated private support, security, or conduct reporting channel**.

That means:

- the repository is **not yet ready for safe private disclosure through an in-repo contact path**
- if a report requires private handling, do not post sensitive details publicly
- until maintainers publish a real private route, this repository can only document the boundary, not fully solve it
