# Commit Message Request

Copy this template into your chat with OpenCode to generate a structured, conventional commit message based on your staged changes.

## 📝 Request

Please review my staged git changes and generate a conventional commit message.

### 🎯 Goal
Create a clear, concise commit message following the Conventional Commits specification.

### 📋 Constraints
- Use the `<type>(<scope>): <subject>` format.
- The subject line must be 50 characters or less.
- The subject line must be in the imperative mood (e.g., "add feature", not "added feature").
- If necessary, include a blank line followed by a detailed body explaining *why* the change was made (not *what* was changed, the diff shows that).
- Do not invent ticket numbers if none are provided.

### 📂 Context
- [Optional: Mention any specific issue number or context, e.g., "Fixes #123", "Related to the new login flow"]

### 📤 Output Format
Provide only the final commit message in a code block, ready to be copied.
